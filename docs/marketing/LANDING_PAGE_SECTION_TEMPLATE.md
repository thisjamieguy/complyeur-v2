# Landing Page Section Template

Last updated: 2026-06-23

Use this document to collect landing-page inspiration, decide the page structure,
and turn screenshots from other sites into a clear redesign brief for ComplyEur
or future product pages.

This is a template. Duplicate it before a new redesign and fill in the TODO
fields.

## Screenshot Workflow

Store local screenshots in:

```text
docs/marketing/landing-page-screenshots/
```

Recommended naming:

```text
01-hero-source-name.png
02-social-proof-source-name.png
03-problem-source-name.png
```

To add a screenshot, replace the placeholder line in each section with:

```md
![Hero inspiration](landing-page-screenshots/01-hero-source-name.png)
```

For each screenshot, capture the idea rather than copying the page exactly.
Record what to adapt, what to avoid, and why it suits ComplyEur.

## Recommended Page Order

This is the default order for a B2B SaaS landing page. Remove optional sections
only when they do not serve the page goal.

| Order | Section | Required? | Purpose |
| --- | --- | --- | --- |
| 1 | Hero | Yes | State the outcome, audience, and primary CTA |
| 2 | Trust Strip | Yes | Show immediate credibility with logos, badges, numbers, or regulatory relevance |
| 3 | Problem | Yes | Name the painful current state the buyer already recognizes |
| 4 | Stakes | Recommended | Show what happens if the problem is ignored |
| 5 | Solution Overview | Yes | Explain the product category and why it is different |
| 6 | Product Preview | Yes | Show the actual product or workflow outcome |
| 7 | Feature Blocks | Yes | Explain the 3 to 5 capabilities that support the promise |
| 8 | How It Works | Yes | Reduce perceived setup complexity |
| 9 | Role Or Use-Case Paths | Recommended | Help different buyers find themselves on the page |
| 10 | Compliance And Trust | Yes for ComplyEur | Address security, privacy, RLS, GDPR, and compliance confidence |
| 11 | Integrations And Data Import | Recommended | Reduce migration and operational friction |
| 12 | Proof Section | Recommended | Add testimonials, case studies, quantified outcomes, or pilot results |
| 13 | Pricing Or Demo Offer | Recommended | Clarify the buying motion and next step |
| 14 | FAQ | Yes | Handle objections before the final CTA |
| 15 | Final CTA | Yes | Give a clean next step after the argument is complete |
| 16 | Footer | Yes | Include legal, product, company, and contact links |
