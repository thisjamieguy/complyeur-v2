# DPIA Trigger Checklist

Last reviewed: 2026-06-24

Use this checklist before shipping changes that could materially increase privacy
risk. If any trigger is marked yes, complete a DPIA review before release.

## Triggers

| Question | Yes / No | Notes |
| --- | --- | --- |
| Does the change introduce automated decision-making or profiling that materially affects an individual? |  |  |
| Does it add new employee-level scoring, risk ranking, prediction, or monitoring beyond current Schengen compliance calculations? |  |  |
| Does it process special-category, criminal-offence, or children's data? |  |  |
| Does it introduce large-scale processing beyond current customer/workspace scope? |  |  |
| Does it add a new AI feature using customer, employee, trip, import, support, or billing data? |  |  |
| Does it add a new processor/subprocessor or new international transfer path? |  |  |
| Does it add a new export, reporting surface, API route, webhook, or integration containing personal data? |  |  |
| Does it make personal data available to a new role, team, customer, or administrator class? |  |  |
| Does it materially change retention, deletion, anonymisation, or DSAR behavior? |  |  |
| Does it introduce new tracking, analytics, cookies, local storage, or marketing pixels? |  |  |

## Minimum DPIA Output

- Processing purpose and necessity.
- Data-subject categories and data categories.
- Risk assessment for confidentiality, fairness, transparency, and rights.
- Mitigations, alternatives considered, and residual risk.
- Processor/transfer review.
- Security controls and access model.
- Retention and DSR impact.
- Decision owner and legal/privacy sign-off.
